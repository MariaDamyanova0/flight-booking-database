# Flight Booking Database System

## Overview

This project demonstrates the design and implementation of a relational database for a flight booking system.

The database models the core components of an airline reservation platform, including flights, passengers, bookings, and related data relationships.

The goal of the project was to practice relational database design, normalization, and SQL querying within a realistic booking system.

---

## Features

- Relational database structure
- Multiple interconnected tables
- Data integrity using primary and foreign keys
- SQL queries for managing flights and reservations
- Structured relationships between passengers, flights, and bookings

---

## Database Design

The database models a simplified airline reservation system including:

- Passengers
- Flights
- Bookings
- Airports or routes
- Flight schedules

These entities are connected through relational keys to ensure consistent and reliable data management.

---

## Technologies Used

- SQL
- Relational Database Design
- Data Normalization
- Query Design

---

## Purpose

This project explores how structured databases support real-world reservation systems such as airline booking platforms.
